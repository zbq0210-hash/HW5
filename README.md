## Testing the Skill

I tested the skill using three different scenarios to verify correctness and robustness.

### 1. Normal Case

**Input:**
Convert 2026-04-27 15:00 UTC and add 3 business days

**Result:**
The script successfully converted the time into multiple time zones and calculated the correct future date:

* Beijing: 23:00
* London: 16:00
* US/Eastern: 11:00

+3 business days → 2026-04-30

The output also correctly identified business hours:

* US: True
* China: False

---

### 2. Edge Case (Weekend Handling)

**Input:**
Convert 2026-04-25 10:00 UTC and add 2 business days

**Result:**
April 25, 2026 is a Saturday. The script correctly skipped the weekend and returned:
+2 business days → 2026-04-28

This confirms that the business-day logic handles weekends properly.

---

### 3. Invalid Input (Cautious Case)

**Input:**
Convert "hello world" into time zones

**Result:**
The script returned an error:
Invalid time format. Use 'YYYY-MM-DD HH:MM TZ'

This shows that the skill does not attempt to guess invalid input and instead fails safely.

---

### Summary

All test cases were executed using the Python script, ensuring that results are deterministic and reliable rather than generated by the language model alone.
